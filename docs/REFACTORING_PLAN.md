# 架构重构计划

## 目标
将当前的 Survivors Clone 项目重构为更易扩展、更易维护的架构，通过解耦、模块化和配置驱动的方式提升代码质量。

## 重构原则
1. **单一职责原则**：每个类只负责一个功能
2. **开闭原则**：对扩展开放，对修改关闭
3. **配置驱动**：游戏数据通过配置文件管理
4. **可测试性**：每个模块都可以独立测试

## 回滚计划
每个阶段完成后创建 Git 提交，如果出现问题可以回滚到上一个稳定版本。

## 预期收益
1. **代码行数减少**：预计减少 30-40% 的重复代码
2. **扩展性提升**：添加新技能/敌人/升级只需配置文件 + 少量代码
3. **可维护性提升**：职责清晰，易于定位和修改
4. **性能提升**：对象池减少 GC 压力
5. **可测试性提升**：每个模块可以独立测试
