## 🏗️ 架构设计

### 输入处理层级

```
┌─────────────────────────────────────┐
│         World.gd (全局输入)          │
│  - 宗派切换（1-4）                   │
│  - 武器切换（5-0）                   │
│  - 使用_input()捕获按键事件          │
└─────────────────────────────────────┘
                 ↓
┌─────────────────────────────────────┐
│      InputManager (游戏输入)         │
│  - 移动（WASD）                      │
│  - 攻击（空格/左键）                 │
│  - 冲刺（Shift）                     │
│  - 技能（Q/E/R）                     │
│  - 使用_process()轮询输入状态        │
└─────────────────────────────────────┘
                 ↓
┌─────────────────────────────────────┐
│        各个组件处理信号              │
│  - AttackManager                     │
│  - DashManager                       │
│  - ActiveSkillManager                │
└─────────────────────────────────────┘
```

### 设计原则

1. **分层处理**: 
   - 全局功能（宗派/武器切换）在World层处理
   - 游戏功能（移动/攻击/技能）在InputManager层处理

2. **事件驱动**: 
   - InputManager使用信号机制
   - 组件订阅信号，解耦输入和逻辑

3. **防抖动**: 
   - 使用`key_cooldown`防止重复触发
   - 默认冷却时间：0.3秒

## 🔧 实现细节

### World.gd - 全局输入处理

```gdscript
func _input(event: InputEvent):
    if event is InputEventKey and event.pressed:
        match event.keycode:
            KEY_1: _switch_sect(0)  # 冰心宗
            KEY_2: _switch_sect(1)  # 雷鸣宗
            KEY_3: _switch_sect(2)  # 烈焰宗
            KEY_4: _switch_sect(3)  # 毒瘴宗
            KEY_5: _switch_weapon(0)  # 无名
            KEY_6: _switch_weapon(1)  # 寒川
            # ...
```

**特点**:
- ✅ 使用`_input()`立即响应按键
- ✅ 使用`match`语句清晰映射
- ✅ 自动解锁武器（测试友好）
- ✅ 带冷却时间防止误触

---

### InputManager.gd - 游戏输入处理

```gdscript
func _update_skills():
    if Input.is_action_just_pressed("skill_q"):
        emit_signal("skill_q_pressed")
    
    if Input.is_action_just_pressed("skill_e"):
        emit_signal("skill_e_pressed")
    
    if Input.is_action_just_pressed("skill_r"):
        emit_signal("skill_r_pressed")
```

**特点**:
- ✅ 使用Godot的Action系统
- ✅ 通过信号解耦
- ✅ 在`_process()`中轮询

## 🎮 使用指南

### 快速开始游戏
```bash
# 方式1：直接运行主场景
godot World/world.tscn

# 方式2：使用快速测试脚本
scripts\quick_test.bat
```

### 游戏中切换
1. **切换宗派**: 按数字键1-4
   - 会立即解锁对应宗派的Q/E/R技能
   - UI指示器会高亮显示当前宗派

2. **切换武器**: 按数字键5-0

   - UI指示器会高亮显示当前武器

3. **释放技能**: 按Q/E/R
   - 需要先选择宗派
   - 技能有冷却时间

## 🐛 故障排查

### 问题：按键1-0无响应

**可能原因**:
1. ❌ World脚本未加载
2. ❌ UI遮挡了输入
3. ❌ 按键冷却中
4. ❌ 组件未初始化

**解决方法**:
```bash
# 1. 验证World脚本加载
godot --headless --script tests/test_world_input.gd

# 2. 检查world.tscn是否包含脚本引用
# 应该有: script = ExtResource("15_world")

# 3. 等待0.3秒再按下一个键

# 4. 确保Player已完全初始化
# World._ready()中有await get_tree().process_frame
```

## 📊 性能考虑

### 输入处理性能
- **World._input()**: 事件驱动，零开销（仅在按键时触发）
- **InputManager._process()**: 每帧轮询，约0.01ms/帧
- **按键冷却**: 防止重复触发，减少不必要的处理

### 优化建议
1. ✅ 使用`_input()`处理离散事件（宗派/武器切换）
2. ✅ 使用`_process()`处理连续输入（移动）
3. ✅ 添加冷却时间防止抖动
4. ✅ 使用`match`语句提高可读性
