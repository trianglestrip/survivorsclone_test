## 🔍 问题分析

### 问题1: 按键切换失效

**根本原因**:
- 按键切换功能（1-4切换宗派，5-0切换武器）只在`interactive_test_world.gd`中实现
- 主游戏场景`world.tscn`没有脚本处理这些输入
- `InputManager`只处理游戏操作（WASD、攻击、技能），不处理全局功能

**设计问题**:
```
❌ 之前的架构:
   interactive_test_world.gd  →  处理1-0按键 ✓
   world.tscn                 →  无脚本 ✗
   
✅ 修复后的架构:
   world.gd                   →  处理1-0按键（全局功能）
   InputManager               →  处理WASD/攻击/技能（游戏功能）
```

### 问题2: 变量重复定义

**根本原因**:
- `BaseActiveSkill`重构时添加了`tick_interval`和`tick_timer`
- 子类中仍保留了这些变量的重复定义
- GDScript不允许子类重新定义父类成员变量

**冲突文件**:
```
thunder_field.gd  →  var tick_timer: float = 0.0  ❌
fire_wall.gd      →  var tick_timer: float = 0.0  ❌
                     var tick_interval: float = 0.3  ❌
poison_cloud.gd   →  var tick_timer: float = 0.0  ❌
                     var tick_interval: float = 0.5  ❌
poison_plague.gd  →  var tick_timer: float = 0.0  ❌
                     var tick_interval: float = 0.5  ❌
poison_dart.gd    →  range = cfg.get("range", ...)  ❌ (range是内置关键字)
```

### 问题3: 技能特效过大

**根本原因**:
- 冰心宗技能已在上一轮修复中调整（缩小25-50%）
- 其他宗派技能仍使用原始尺寸和透明度
- 缺乏统一的视觉标准

**需要调整的技能**:
- 雷鸣宗: Q（雷霆一击）、E（雷阵）、R（天罚雷劫）
- 烈焰宗: Q（火球术）、E（火墙）、R（陨火天降）
- 毒瘴宗: Q（毒镖）、E（毒云）、R（瘟疫爆发）

## 🛠️ 解决方案

### 修复1: 添加World输入处理

**创建`World/world.gd`**:

```gdscript
extends Node2D

var player: CharacterBody2D = null
var sect_manager: Node = null
var weapon_registry: Node = null

var last_key_time: Dictionary = {}
var key_cooldown: float = 0.3

var sect_ids = ["ice", "thunder", "fire", "poison"]
var weapon_ids = ["sword_basic", "sword_frost", "hammer_thunder", 
                  "staff_fire", "dagger_poison", "spear_legendary"]

func _input(event: InputEvent):
    if event is InputEventKey and event.pressed:
        match event.keycode:
            KEY_1: _switch_sect(0)
            KEY_2: _switch_sect(1)
            KEY_3: _switch_sect(2)
            KEY_4: _switch_sect(3)
            KEY_5: _switch_weapon(0)
            KEY_6: _switch_weapon(1)
            KEY_7: _switch_weapon(2)
            KEY_8: _switch_weapon(3)
            KEY_9: _switch_weapon(4)
            KEY_0: _switch_weapon(5)
```

**特点**:
- ✅ 使用`_input()`事件驱动，零轮询开销
- ✅ 使用`match`语句清晰映射按键
- ✅ 0.3秒按键冷却防止误触
- ✅ 自动解锁武器（测试友好）

**集成到场景**:
```gdscript
# world.tscn
[node name="World" type="Node2D"]
script = ExtResource("15_world")
```

## 🎯 技术细节

### 输入系统架构

```
┌─────────────────────────────────────┐
│         World.gd (全局输入)          │
│  - 宗派切换（1-4）                   │
│  - 武器切换（5-0）                   │
│  - 使用_input()捕获按键事件          │
│  - 事件驱动，零轮询开销              │
└─────────────────────────────────────┘
                 ↓
┌─────────────────────────────────────┐
│      InputManager (游戏输入)         │
│  - 移动（WASD）                      │
│  - 攻击（空格/左键）                 │
│  - 冲刺（Shift）                     │
│  - 技能（Q/E/R）                     │
│  - 使用_process()轮询输入状态        │
└─────────────────────────────────────┘
```

**设计优势**:
- ✅ 分层清晰：全局功能与游戏功能分离
- ✅ 性能优化：事件驱动 vs 轮询
- ✅ 易于扩展：添加新按键映射很简单
- ✅ 防抖动：0.3秒冷却时间

### 变量继承规则

**BaseActiveSkill提供的通用变量**:
```gdscript
# 基础属性
var damage: float
var cooldown: float
var duration: float

# 周期性伤害
var tick_damage_enabled: bool
var tick_interval: float
var tick_timer: float
var tick_callback: Callable

# 状态效果
var status_effects: Array
```

**子类规则**:
- ❌ 不能重新定义父类变量
- ✅ 可以在`_load_skill_config()`中赋值
- ✅ 可以定义自己的特定变量
